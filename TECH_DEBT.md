# Технический долг

Осознанный список известных ограничений и упрощений. Часть из них — сознательные
компромиссы в рамках тестового задания (чтобы не раздувать объём), часть — реальные
недоработки, которые я вижу и знаю, как чинить. Вынесено отдельно намеренно: важнее
показать зрелое отношение к качеству, чем сделать вид, что долга нет.

Проверено вручную в браузере (dev-сборка) + ревью кода.

## Осознанные компромиссы (сделаны намеренно)

- **Потеря позиции скролла и подгруженного контента при возврате из карточки.**
  При переходе на `/game/:id` и обратно счётчик `loadedCount` сбрасывается на первую
  страницу, а позиция скролла не восстанавливается. Воспроизведено: высота документа
  падает `18693px → 4006px`, скролл `~15800 → ~1141`. Данные каталога при этом остаются
  в кэше RTK Query — теряется только «сколько подгружено» и позиция.
  _Как чинить:_ хранить `loadedCount` + scroll в сторе/URL и восстанавливать при возврате
  (или `ScrollRestoration` React Router). Не стал усложнять тестовое.

- **Фильтры не синхронизированы с URL.** Поиск и тип живут только в Redux, поэтому
  состояние нельзя расшарить ссылкой, а кнопка «назад» браузера их не отматывает.
  _Как чинить:_ вынести фильтры в query-параметры (связано с пунктом выше).

## UI / UX и вёрстка

- **Хрупкая мобильная вёрстка строки поиска.** В `.field` (`GameSearch.module.scss`)
  задана фиксированная `width: 356px`, а `.row` — flex без переноса. На ровно 375px
  вписывается впритык, но на чуть более узких/пограничных вьюпортах кнопка `SEARCH`
  прижимается к краю и обрезается (воспроизводилось на ~381px).
  _Как чинить:_ убрать фиксированную ширину, добавить `flex-wrap` / адаптив кнопки.

- **Непереведённый тип игры в фильтре.** Код `cs` отсутствует в `GAME_TYPE_LABELS`
  (`entities/game/lib/gameTypes.ts`), поэтому в селекте показывается сырой `CS` вместо
  человекочитаемого названия.
  _Как чинить:_ добавить лейбл (или более аккуратный фолбэк).

## Код и архитектура

- **Детальная страница тянет весь каталог.** При прямом заходе/refresh на `/game/:id`
  грузятся все 1017 игр ради одной (в API нет эндпоинта на конкретную игру). Приемлемо
  сейчас (переиспользуется кэш), но не масштабируется.
  _Как чинить:_ отдельный запрос по id, если/когда API его поддержит.

- **Хрупкая связка геометрии JS ↔ CSS.** `captionHeight` и `aspectRatio` в
  `widgets/games-catalog/config.ts` синхронизируются с SCSS-токенами вручную. При смене
  шрифта/отступов виртуализатор рассчитает высоту строки неверно.
  _Как чинить:_ единый источник (CSS-переменные, читаемые из JS) или измерение по факту.

- **Неиспользуемое поле модели.** `firstSeenAt` присутствует в типе `Game`, но нигде не
  используется (нет сортировки/бейджа «новинки»).

- **Нет глобального Error Boundary.** Непойманная ошибка рендера уронит всё приложение
  вместо локального fallback-экрана.

## Тестирование и инфраструктура

- **Не покрыты тестами хуки и интеграция.** Есть юнит-тесты на чистую логику
  (`filterGames`, утилиты, `GameCard`), но нет тестов на `useInfiniteScroll` /
  `useWindowVirtualizer` и на интеграцию виджета каталога; нет E2E (Playwright).

- **Прод-деплой не настроен.** Планируется GitHub Pages через Actions. Ограничение
  статического хостинга (нет серверного прокси для `/api` при отсутствии CORS у апстрима
  и раздача по субпути `/<repo>/`) нужно решить отдельно — см. README, раздел «Деплой».
